# 时间选择器功能设计文档

> 日期：2026-06-18
> 项目：LeaveApp（请假系统安卓应用）

---

## 一、概述

为请假系统 Android App 增加滚轮式时间选择功能，替代原有的手动输入时间方式。同时将编辑功能统一整合到"提交续假"入口，删除原有的卡片点击编辑功能。

### 核心原则

- **不修改已有页面的外观** — 列表页面（顶栏、卡片、丝带标签、底部栏、VPN标签、颜色、字体等）完全保持不变
- **只添加功能，不修改已确定的 UI**
- 新增的"提交续假"操作页面可以自由设计，以便于操作

## 三、交互流程

```
用户打开 App
  → 看到日常请假列表（三条记录，排序：待休假置顶 → 其余按 id 倒序）
  → 点击底部"提交续假"按钮
    → 从右侧滑入全屏编辑页面
    → 页面预填充**列表中排序后的第一条记录**的所有字段
    → 注意：按钮文字"提交续假"保持不动，行为是编辑第一条记录
    → 用户修改字段：
      - 学号、姓名 → 输入框直接编辑
      - 开始时间 → 点击 → DatePicker 日历选择 → 确定 → TimePicker 时/分滚轮 → 确定
      - 结束时间 → 点击 → DatePicker 日历选择 → 确定 → TimePicker 时/分滚轮 → 确定
      - 请假时长 → 自动计算（根据开始和结束时间）
      - 是否续假 → 勾选框
      - 请假原因 → 输入框
      - 销假状态 → 输入框
      - 状态标签 → 输入框
    → 点击页面顶部右上角\"保存\" → 更新 Room 数据库 → 自动返回列表页 → 列表刷新\n    → 点击页面顶部左上角\"返回\" → 返回列表页，不保存
```

```

├──────────────────────────────┤
│                              │
│  学号  [________________]    │
│  姓名  [________________]    │
│                              │
│  开始时间  [点击选择 >]      │  ← 点击后弹出 DatePicker → TimePicker
│  结束时间  [点击选择 >]      │  ← 同上
│  请假时长  [自动计算显示]    │  ← 只读显示，不可编辑
│                              │
│  是否续假  [  ] 是           │  ← Checkbox
│                              │

│                              │

```

### 4.2 时间选择流程

点击"开始时间"或"结束时间"行时的交互：

```
点击时间行
  → DatePicker 以当前记录的日期为默认值打开
  → 用户选择日期，点击"确定"
  → DatePicker 关闭
  → TimePicker（时/分滚轮）弹出
  → 用户选择小时和分钟，点击"确定"
  → TimePicker 关闭
  → 时间字段显示为 "YYYY-MM-DD HH:mm" 格式
  → 自动重新计算请假时长
```

### 4.3 请假时长自动计算逻辑

```
输入：开始时间（startTime），结束时间（endTime）
输出：格式如 "0天3小时"、"1天2小时"

算法：
1. 解析 startTime 和 endTime 为 LocalDateTime
2. 计算 Duration.between(start, end)
3. 提取总小时数 → 天 = 总小时 / 24，剩余小时 = 总小时 % 24
4. 格式化输出 "X天Y小时"

边界情况：
- 结束时间早于开始时间 → 显示"时间有误"
- 结束时间等于开始时间 → 显示"0天0小时"
```

### 4.4 保存逻辑

点击页面顶部右上角"保存"文字或图标时：
1. 收集页面上所有字段的当前值
2. 调用 `LeaveViewModel.updateLeave(updatedRecord)`
3. ViewModel 调用 `LeaveRepository.updateLeave()`
4. Repository 调用 `LeaveDao.updateLeave()`
5. Room 数据库更新记录
6. 全屏页面关闭，返回列表页
7. LazyColumn 自动刷新（通过 Flow 观察）

## 六、导航方式

由于项目目前是单页（`LeaveListScreen` 直接设置在 `MainActivity` 中），导航到全屏编辑页面有两种方式：

**方式 A：在 Activity 层面管理** — `MainActivity` 中使用 `mutableStateOf` 控制显示哪个页面，列表页和编辑页是平级切换

**方式 B：使用 Compose Navigation** — 引入 `androidx.navigation` 库进行页面路由

**选择：方式 A**（最简单，不引入额外依赖，符合项目当前轻量结构）

## 七、注意事项

1. **不修改列表页外观** — 所有改动仅限于新增的编辑页面和底层逻辑清理
2. **时间显示格式** — 时间字段在编辑页和列表页的显示格式保持一致（"YYYY-MM-DD HH:mm"）
3. **首次打开编辑页** — 预填充列表中排序后第一条记录的当前值
4. **\"是否续假\"字段** — 手动勾选，不自动判断
5. **没有待休假记录时** — 仍然编辑列表最上面（第一条）的记录
6. **保存位于页面顶部右上角** — 返回位于顶部左上角，底部不放操作按钮
7. **数据一致性** — 保存后列表页自动刷新，不需要手动刷新
